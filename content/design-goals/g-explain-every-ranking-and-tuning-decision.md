## Goal

For every decision Agent Studio makes or proposes — a ranking strategy choice, a config recommendation, a result reordering, an agent action — **the reason is a first-class output of the same call**, not a separate log lookup.

This is a customer trust requirement, not a UX polish item. The analytics-assistant finding is explicit: suggestions without reasoning don't generate buyer confidence. The same logic applies to runtime agent decisions.

## What "achieving this goal" looks like

- Every Agent Studio response includes structured reasoning data the customer can render, store, or pipe into their own systems.
- Customers can build a Merchandising-Studio-style "click on a result to see why it ranked here" surface from our primitives, without a separate explain API.
- Reasoning is generated in the same call as the decision (not retroactively summarized), so it reflects the actual decision path.

## What this rules out

- Logs-as-explainability: a separate "debug mode" doesn't satisfy the trust requirement, because customers can't put it in front of *their* business stakeholders.
- Plausible-sounding post-hoc explanations generated by an LLM after the ranking decision was made — that's reasoning theater and will erode trust when it diverges from the real cause.
- Tracking reasoning only for "important" decisions. Every customer-perceived decision needs reasoning; we can't predict which ones will get challenged.
